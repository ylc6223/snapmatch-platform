# SnapMatch 开发任务清单

本目录用于跟踪 SnapMatch 平台各应用的开发进度和任务完成情况。

## 📁 目录结构

```
checklist/
├── README.md              # 本文档
├── admin/                 # 管理后台任务清单
│   └── implementation-status.md
├── web/                   # 官网任务清单
│   └── README.md
└── backend/               # 后端服务任务清单
    └── README.md
```

> **说明**: 小程序项目因技术栈原因独立在其它仓库中，不在本 checklist 范围内。

## 📝 使用说明

### 管理后台 (Admin)

管理后台的任务清单位于 `checklist/admin/implementation-status.md`，包含：

- ✅ 已实现功能
- ❌ P0 核心功能
- ⚠️ P1 重要功能
- 📊 P2 增强功能

### Web 端 (官网)

Web 端（官网）的任务清单位于 `checklist/web/`。

> **注意**: 小程序项目因技术栈原因在其它仓库中管理

### 后端服务 (Backend)

后端服务的任务清单位于 `checklist/backend/`。

## 🎯 标记规范

使用以下格式标记任务状态：

```markdown
# 未开始

- [ ] 任务名称

# 已完成

- [x] 任务名称

# 带注释的完成

- [x] 任务名称 # 完成于 2026-01-10

# 部分完成

- [ ] 任务名称 🟨（前端已完成，后端接口开发中）

# 已取消

- [ ] 任务名称 🚫（需求变更，不再需要）
```

## 📊 进度统计

各应用的总体进度会在各自目录的文档顶部显示。
