## 🎯 设计原则

### 通用原则

- **清晰优先** - 设计应该让用户一目了然，减少认知负担
- **一致性** - 视觉风格、交互模式保持一致
- **效率至上** - 特别是 B 端产品，注重操作效率
- **反馈及时** - 每个操作都应该有明确的反馈
- **容错性** - 预防错误，并提供恢复机制

### Admin 特有原则

- **信息密度** - 在有限空间展示足够多的信息
- **快速操作** - 减少操作步骤，提供快捷方式
- **数据可视化** - 用图表展示复杂数据
- **权限感知** - 根据角色展示不同界面

### Web 特有原则

- **视觉冲击** - 展示高质量摄影作品
- **沉浸体验** - 突出内容，减少干扰
- **移动优先** - 响应式设计，适配各种设备
- **SEO 友好** - 利于搜索引擎优化

## 📝 文档编写规范

### 页面设计文档应包含

1. **页面概述** - 页面的用途和目标
2. **用户场景** - 谁在使用这个页面，为什么使用
3. **信息架构** - 页面的结构和内容层级
4. **交互流程** - 用户如何完成核心任务
5. **设计决策** - 为什么这么设计，考虑了哪些因素
6. **响应式考虑** - 不同屏幕尺寸的适配
7. **状态说明** - 各种状态（空态、加载态、错误态）的处理
8. **可访问性** - 无障碍设计考虑

### 组件设计文档应包含

1. **组件概述** - 组件的用途
2. **视觉设计** - 尺寸、颜色、字体、间距
3. **交互行为** - 交互状态和反馈
4. **使用场景** - 在哪些地方使用
5. **变体说明** - 不同变体的用途
6. **代码示例** - 使用示例（可选）

## 📋 设计流程

### 1. 需求分析

- 理解业务目标和用户需求
- 分析竞品和参考案例
- 确定设计约束

### 2. 信息架构

- 梳理页面结构和内容层级
- 设计用户流程
- 确定交互模式

### 3. 视觉设计

- 设计页面布局
- 确定视觉风格
- 制作高保真原型

### 4. 设计评审

- 与团队评审设计方案
- 收集反馈并迭代
- 获得最终确认

### 5. 开发对接

- 输出设计规范文档
- 与开发者对接实现细节
- 跟进开发还原度
