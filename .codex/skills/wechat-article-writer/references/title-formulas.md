# Title Formulas Library

> Proven patterns and headline strategies for WeChat Official Account articles

---

## Title Principles

### Core Guidelines

1. **Length: 15-25 Characters**
   - Too short: Lacks information
   - Too long: Gets cut off on mobile

2. **Balance Curiosity with Clarity**
   - Curiosity draws clicks
   - Clarity sets expectations
   - Avoid pure clickbait

3. **Front-Load Value**
   - Put key terms early
   - WeChat may truncate long titles
   - Mobile screens show ~20 chars

4. **Avoid Forbidden Patterns**
   - ❌ "震惊！"、"必看！"、"颠覆性"
   - ❌ Anxiety-inducing: "再不学就晚了"
   - ❌ Overpromising: "学会这个技巧月入百万"

## Formula Categories

### 1. Problem-Oriented Titles

**Pattern A: "为什么 [X] 没你想的那么简单"**

**Use When:** Addressing common misconceptions

**Examples:**

- 为什么 AI 代码补全没你想的那么简单
- 为什么 RSC 不是 SSR 的升级版
- 为什么 Prompt Engineering 不是玄学

**Structure:**

```
为什么 [技术名词] [反直觉观点]
```

**Pattern B: "[X] 的真正难点在哪里"**

**Use When:** Deep-diving into technical challenges

**Examples:**

- Cursor 的真正难点在哪里
- React Server Components 的真正难点在哪里
- AI 工具落地的真正难点在哪里

**Pattern C: "[常见问题]，本质是 [根因] 问题"**

**Use When:** Reframing common problems

**Examples:**

- AI 补全不准，本质是 Context 问题
- 前端性能差，本质是架构问题
- 代码难维护，本质是抽象问题

---

### 2. Solution-Oriented Titles

**Pattern A: "[X] 的三种工程实现方式"**

**Use When:** Comparing implementation approaches

**Examples:**

- AI 对话的三种工程实现方式
- 状态管理的三种工程实现方式
- 代码生成的三种工程实现方式

**Structure:**

```
[功能/需求] 的 [数字] 种 [实现维度]
```

**Pattern B: "如何在 [场景] 中正确使用 [技术]"**

**Use When:** Providing practical guidance

**Examples:**

- 如何在生产环境中正确使用 AI 工具
- 如何在大型项目中正确使用 TypeScript
- 如何在团队中正确使用 Code Review

**Pattern C: "从 [起点] 到 [终点]：[技术] 实践指南"**

**Use When:** Documenting implementation journey

**Examples:**

- 从 Copilot 到 Cursor：AI 工具升级实践
- 从 JS 到 TS：类型系统迁移指南
- 从单体到微前端：架构演进实录

---

### 3. Insight-Oriented Titles

**Pattern A: "理解 [X] 的本质：不是 [Y]，而是 [Z]"**

**Use When:** Correcting fundamental misunderstandings

**Examples:**

- 理解 Prompt Engineering 的本质：不是调参，而是接口设计
- 理解 RSC 的本质：不是 SSR，而是组件级渲染
- 理解 AI 工具的本质：不是替代，而是辅助

**Structure:**

```
理解 [技术] 的本质：不是 [误区]，而是 [正解]
```

**Pattern B: "[X]，被误解最深的 [Y]"**

**Use When:** Addressing widespread misconceptions

**Examples:**

- TypeScript，被误解最深的类型系统
- Server Components，被误解最深的 React 特性
- AI Coding，被误解最深的开发工具

**Pattern C: "换个角度看 [X]：[新视角]"**

**Use When:** Offering fresh perspective

**Examples:**

- 换个角度看 AI 工具：不是效率问题，是思维问题
- 换个角度看前端架构：不是技术选型，是组织设计
- 换个角度看代码审查：不是找错，是传承经验

---

### 4. Comparison-Oriented Titles

**Pattern A: "[X] vs [Y]：工程视角的选择"**

**Use When:** Comparing tools or approaches

**Examples:**

- Cursor vs GitHub Copilot：工程视角的选择
- RSC vs SSR：工程视角的选择
- TypeScript vs JavaScript：工程视角的选择

**Structure:**

```
[选项 A] vs [选项 B]：[评判维度] 的选择
```

**Pattern B: "从 [X] 迁移到 [Y]，值得吗？"**

**Use When:** Evaluating migration decisions

**Examples:**

- 从 Copilot 迁移到 Cursor，值得吗？
- 从 JS 迁移到 TS，值得吗？
- 从 Webpack 迁移到 Vite，值得吗？

**Pattern C: "[X] 和 [Y] 的区别，不只是 [表面]"**

**Use When:** Going beyond surface-level differences

**Examples:**

- Cursor 和 Copilot 的区别，不只是补全速度
- RSC 和 SSR 的区别，不只是渲染位置
- Hooks 和 Class 的区别，不只是写法

---

### 5. Experience-Oriented Titles

**Pattern A: "用了 [X] 半年，我的 [Y] 个观察"**

**Use When:** Sharing long-term usage insights

**Examples:**

- 用了 Cursor 半年，我的 5 个观察
- 用了 AI 工具半年，我的 3 个反思
- 用了 TypeScript 一年，我的 4 个体会

**Structure:**

```
用了 [工具/技术] [时长]，我的 [数字] 个 [输出类型]
```

**Pattern B: "[X] 踩坑实录：[Y] 个教训"**

**Use When:** Documenting failures and learnings

**Examples:**

- AI 代码生成踩坑实录：3 个惨痛教训
- 大型重构踩坑实录：5 个血泪教训
- 微前端踩坑实录：7 个避坑指南

**Pattern C: "在 [场景] 用 [技术]，我后悔了"**

**Use When:** Warning about misapplied technologies

**Examples:**

- 在小项目用微前端，我后悔了
- 在原型阶段用 TypeScript，我后悔了
- 在简单场景用 AI 工具，我后悔了

---

### 6. Trend-Analysis Titles

**Pattern A: "[X] 一年后，我们学到了什么"**

**Use When:** Reflecting on technology adoption

**Examples:**

- AI 编程工具一年后，我们学到了什么
- React 18 一年后，我们学到了什么
- ChatGPT 一年后，我们学到了什么

**Pattern B: "关于 [X]，没人告诉你的 [Y] 件事"**

**Use When:** Revealing overlooked aspects

**Examples:**

- 关于 AI 工具，没人告诉你的 3 件事
- 关于 RSC，没人告诉你的 5 个坑
- 关于前端架构，没人告诉你的 4 个真相

**Pattern C: "[X] 的下一步是什么"**

**Use When:** Forward-looking analysis (use cautiously)

**Examples:**

- AI 编程工具的下一步是什么
- 前端框架的下一步是什么
- 全栈开发的下一步是什么

## Title Testing Checklist

Before finalizing a title, verify:

**Length:**

- [ ] 15-25 characters (optimal for mobile)
- [ ] Key terms front-loaded

**Clarity:**

- [ ] Reader knows what to expect
- [ ] Specific enough to set scope
- [ ] Not overly vague or generic

**Curiosity:**

- [ ] Poses interesting question or perspective
- [ ] Promises valuable insight
- [ ] Avoids clickbait tactics

**Brand Alignment:**

- [ ] No anxiety-inducing language
- [ ] No hype or hyperbole
- [ ] Maintains engineering perspective

**Forbidden Patterns Check:**

- [ ] No "震惊"、"必看"、"颠覆"
- [ ] No "再不学就晚了"
- [ ] No overpromising results

## Title Optimization Process

### Step 1: Generate 5-7 Variations

Use different formula categories:

1. Problem-oriented
2. Solution-oriented
3. Insight-oriented
4. Comparison-oriented
5. Experience-oriented

### Step 2: Test Against Criteria

For each title, score:

- Clarity (1-5)
- Curiosity (1-5)
- Brand fit (1-5)

### Step 3: Select Top 3

Present top 3 to user with rationale:

```markdown
【标题选项】

1. React Server Components：前端架构的新可能
   理由：平衡了新颖性和准确性，避免过度承诺

2. RSC 不是银弹，但值得理解
   理由：诚实的立场，符合品牌调性

3. 从工程视角拆解 React Server Components
   理由：强调工程视角，明确内容定位
```

### Step 4: Iterate Based on Feedback

- Adjust tone if needed
- Refine specificity
- Test different angles

## Anti-Patterns to Avoid

### Clickbait

```
❌ 震惊！AI 工具竟然能做到这个！
❌ 90% 的前端工程师都不知道的秘密
❌ 这个技巧让我的效率提升 10 倍！
```

### Anxiety-Inducing

```
❌ 2024 年还不会 AI 编程的前端工程师
❌ 再不学 TypeScript 就被淘汰了
❌ 你的同龄人已经在用 Cursor 了
```

### Overpromising

```
❌ 掌握 Cursor 让你成为 10x 工程师
❌ 一文学会 React Server Components
❌ AI 工具完全取代传统开发
```

### Too Generic

```
❌ 聊聊 AI 工具
❌ React 新特性介绍
❌ 前端开发的一些思考
```

## Final Title Selection Criteria

Choose the title that:

1. **Accurately represents content**
2. **Engages target audience** (frontend devs interested in AI)
3. **Aligns with brand voice** (rational, engineering-focused)
4. **Avoids hype and anxiety**
5. **Delivers on its promise**

Remember: A good title sets honest expectations and attracts the right readers.
