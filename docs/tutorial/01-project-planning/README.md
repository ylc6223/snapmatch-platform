# 第一阶段：项目规划与架构设计

> **学习目标**：理解项目需求，完成技术选型和架构设计  
> **预计时长**：2-3 小时  
> **难度等级**：⭐⭐☆☆☆

## 本阶段内容

### 1. [需求分析](./01-requirements-analysis.md)

**大纲**：

- 业务背景与用户群体
- 核心功能需求
- 非功能需求（性能、安全、可扩展性）
- 竞品分析
- MVP 范围定义

**学习成果**：能够撰写完整的需求文档

---

### 2. [技术栈选型](./02-tech-stack-selection.md)

**大纲**：

- 前端技术选型（Next.js vs React vs Vue）
- 后端技术选型（NestJS vs Express vs Fastify）
- 数据库选型（MySQL vs PostgreSQL vs MongoDB）
- 存储方案选型（Cloudflare R2 vs 七牛云 vs 腾讯云 COS）
- 技术栈对比与决策依据

**学习成果**：理解技术选型的考量因素

---

### 3. [架构设计](./03-architecture-design.md)

**大纲**：

- 系统架构图（分层架构）
- 模块划分与职责
- 接口设计规范（RESTful）
- 数据流向图
- 安全架构设计（认证、授权、数据加密）

**学习成果**：能够设计清晰的系统架构

---

### 4. [数据库设计](./04-database-design.md)

**大纲**：

- ER 图设计
- 核心表结构设计
  - 用户与权限（RBAC）
  - 作品集管理
  - 照片交付
- 索引策略
- 数据迁移方案

**学习成果**：掌握数据库设计最佳实践

---

### 5. [项目结构规划](./05-project-structure.md)

**大纲**：

- Monorepo vs Multi-repo
- 目录结构设计
  - apps/backend
  - apps/admin
  - apps/web
- 代码组织规范
- 配置文件管理

**学习成果**：规划可维护的项目结构

---
