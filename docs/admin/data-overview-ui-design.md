## 设计规范

### DESIGN SPECIFICATION

**1. Purpose Statement**
为摄影师/管理员提供一个高效的数据监控工作台，快速掌握项目进度、选片状态、待办事项等核心业务指标。基于现有 shadcn/ui 设计系统，保持视觉一致性的同时通过创新布局创造独特的用户体验。

**2. Aesthetic Direction**
**Neutral Dark Industrial** - 中性色深色工业风
基于现有 neutral 色系，融入工业风的精密感。参考线性设计和瑞士国际主义风格，强调网格系统、信息层级和功能性美学。

**3. Color Palette**

```css
/* 使用现有设计系统的颜色变量 */
主色调：
--background: oklch(0.145 0 0)    /* 深黑背景 */
--card: oklch(0.205 0 0)          /* 卡片背景 */
--border: oklch(1 0 0 / 10%)      /* 边框（半透明白） */

强调色：
--lumina-primary: #00cae0        /* 青色主色 */
--lumina-primary-hover: #00a4c2  /* 青色悬停 */
--lumina-amber: #ffbe4d          /* 琥珀色警告 */
--lumina-retouch: #00cae0        /* 精修标记色 */

功能色：
--destructive: oklch(0.704 0.191 22.216) /* 红色警告 */
--chart-1: oklch(0.646 0.222 41.116)      /* 橙色图表 */
--chart-2: oklch(0.6 0.118 184.704)       /* 蓝色图表 */
--chart-3: oklch(0.398 0.07 227.392)      /* 紫色图表 */
```

**4. Typography**

```css
/* 使用现有 Geist 字体系列 */
标题字体：Geist Sans (Variable)
数字字体：Geist Mono (等宽)
正文字体：Geist Sans (Regular/Medium)

字重层级：
- H1: Geist Sans 700 / 48px
- H2: Geist Sans 600 / 32px
- H3: Geist Sans 600 / 24px
- Body: Geist Sans 400 / 16px
- Caption: Geist Sans 400 / 14px
- Number: Geist Mono 600 / 48-72px (数据展示)
```

**5. Layout Strategy**
**"三栏式数据流"布局**

- 左侧固定栏（320px）：待办事项瀑布流
- 中间主内容区（flex-1）：数据卡片网格（3列布局）
- 右侧趋势图区（400px）：固定宽度的趋势图表
- 采用 CSS Grid + Flexbox 混合布局
- 卡片间距：24px（1.5rem）
- 边框圆角：0.625rem（跟随设计系统）

### 布局特点

1. **左侧固定栏** (320px)
   - 垂直滚动，显示所有待办事项
   - 分类：待上传精修、待客户确认、选片即将到期、新订单提醒

2. **中间数据区** (flex-1)
   - 3列网格布局
   - 每个卡片宽度相等，自适应容器宽度
   - 卡片高度：最小 200px，内容自适应

3. **响应式设计**
   - Desktop (>1440px): 3列布局
   - Tablet (768px-1440px): 2列布局
   - Mobile (<768px): 1列布局

## 响应式设计

### 断点定义

页面采用 Mobile First 响应式策略，包含三个主要断点：

- **Mobile** (<768px): 单列布局，侧边栏隐藏
- **Tablet** (768px - 1439px): 2列数据卡片区，侧边栏改为抽屉式
- **Desktop** (≥1440px): 3列数据卡片区，侧边栏固定显示

### 布局适配

| 屏幕尺寸                | 左侧待办栏       | 数据卡片区 | 趋势图区         |
| ----------------------- | ---------------- | ---------- | ---------------- |
| **Mobile** (<768px)     | 隐藏（汉堡菜单） | 1列堆叠    | 全宽，高度 300px |
| **Tablet** (768-1439px) | 抽屉式侧边栏     | 2列网格    | 全宽，高度 350px |
| **Desktop** (≥1440px)   | 固定显示         | 3列网格    | 全宽，高度 400px |

## 交互设计细节

### 1. 卡片交互

**悬停效果**:

- 卡片轻微上浮 4px
- 阴影加深，边框颜色变为青色强调色
- 过渡动画时长 300ms，使用缓动曲线

**点击效果**:

- 项目统计卡片：跳转到项目列表页（按状态筛选）
- 选片进度卡片：展开显示详细进度列表
- 照片数据卡片：跳转到存储管理页面
- 作品集数据卡片：跳转到作品集管理页面

---

### 2. 待办事项交互

**任务项操作**:

- 左侧复选框：快速标记为完成
- 点击任务主体：跳转到对应项目详情
- 长按任务：显示快捷菜单（标记为紧急/推迟/删除）

**快捷菜单**:

- 标记为紧急
- 推迟到明天
- 分配给他人

---

### 3. 页面加载动画

**加载顺序**（延迟进入动画）:

1. **Header** (0ms): 从上方滑入淡入
2. **左侧待办栏** (100ms): 从左侧滑入淡入
3. **数据卡片** (200-500ms): 逐个缩放淡入，每个卡片延迟 100ms
4. **趋势图** (600ms): 从下方升入淡入

动画时长：500ms，使用缓动曲线

---

### 4. 数据刷新交互

**自动刷新**:

- 间隔：每 5 分钟自动刷新一次数据
- 指示器：Header 右上角显示 "最后更新：X 分钟前"
- 手动刷新：点击刷新按钮，显示加载动画

**加载状态**:

- 使用骨架屏组件作为占位符
- 每个卡片显示骨架屏动画
- 趋势图显示加载中的进度条

  - 最近 3 个月
  - 最近 1 年

**数据导出**:

## 设计系统遵循检查

### ✅ 颜色系统

- 使用现有 CSS 变量 (`--background`, `--foreground`, `--card`)
- 使用 neutral 色系作为主色
- 使用 `--lumina-primary` (#00cae0) 作为强调色
- 未使用禁用颜色（紫色/紫罗兰/品红）

### ✅ 字体系统

- 使用 Geist Sans (正文)
- 使用 Geist Mono (数字、等宽文本)
- 未使用禁用字体（Inter, Roboto, Arial）

### ✅ 图标系统

- 使用 Lucide React
- 未使用 emoji 图标

### ✅ 圆角系统

- 使用 `--radius: 0.625rem`
- 未使用直角边框

### ✅ 间距系统

- 使用 Tailwind 间距单位（4px 基准）
- 卡片间距：24px (1.5rem)
- 组件内边距：16px (1rem)
