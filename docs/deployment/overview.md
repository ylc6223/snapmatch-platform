# 部署总览

本项目的部署目标是：**对外一个入口（域名或 IP）**，由网关/反向代理按路径把请求分发到 Web / Admin / Backend。

## 1. 选择哪种部署方式

### 1.1 域名访问（推荐）

- 适用：域名已备案，或计划走标准域名上线
- 入口文档：[域名访问部署](./access.md)

### 1.2 IP 访问（无需备案，含 1Panel）

- 适用：域名暂不可用，需要先用 IP 跑通部署与验收
- 入口文档：[IP 访问部署（含 1Panel）](./ip-access.md)

## 2. 常用验证与排查入口

- 访问异常、服务未更新、CI/CD 失败等：优先看 [部署排查指南](./troubleshooting.md)

## 3. 本地端口（开发约定）

- Admin：`http://localhost:3001`
- Backend：`http://localhost:3002`
