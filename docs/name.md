# 摄影工作室官网与管理后台 GitHub 仓库命名方案

> 本文用于整理并沉淀「摄影工作室官网 + 管理后台」GitHub 仓库的命名思路与推荐方案，整体围绕品牌 **「一拍即合」** 及其小程序与服务体系。

---

## 一、命名背景与目标

当前规划的项目特点：

- 📸 摄影工作室 **官方网站**
- 🧩 内部 **管理后台（Admin / Console）**
- 🔗 服务于同一品牌与小程序 **「一拍即合」**
- 🧑‍💻 面向 GitHub 的工程仓库，而非直接面向 C 端用户

### 命名目标

- 以**品牌为核心**，利于长期维护
- 清晰表达 **Web / Platform / Admin** 等技术属性
- 使用 **英文命名**，更符合 GitHub / 技术社区习惯
- 具备未来扩展空间（多端、多服务）

---

## 二、总体命名原则（建议遵循）

> **品牌名 + 技术/职能后缀**

推荐结构示例：

```text
<brand>-<platform | web | studio | admin | console>
```

说明：

- `brand`：品牌英文名（推荐统一）
- 后缀用于区分官网、后台或整体平台

---

## 三、品牌英文名建议（对应「一拍即合」）

为了便于技术栈与国际化表达，建议为「一拍即合」确定一个稳定的英文品牌名。

### 推荐候选

| 英文名 | 含义与说明 |
|------|-----------|
| **SnapMatch** ⭐ | 最贴合“一拍即合”，产品感强、好扩展（强烈推荐） |
| ShotMatch | 偏专业摄影语境 |
| ClickMatch | 偏轻产品、偏工具 |
| PerfectShot | 偏品牌与营销 |
| OneShot | 有“一次命中 / 一拍即合”之意 |

> **推荐最终品牌英文名：`SnapMatch`**

---

## 四、主推荐仓库命名方案

### 方案 1：官网 + 管理后台在同一仓库（推荐）

```text
snapmatch-platform
```

含义说明：

- `SnapMatch`：品牌名
- `Platform`：承载官网、后台及未来服务

适合：
- 初期项目
- 单仓多应用（Monorepo）

---

### 方案 2：偏 Web 官方站表达

```text
snapmatch-web
```

适合：
- 官网为主
- 后台为子模块

---

### 方案 3：偏工作室品牌表达

```text
snapmatch-studio
```

适合：
- 强调摄影工作室属性
- 未来可能有多个城市 / 分支

---

## 五、官网与管理后台拆分命名（可选）

如果未来拆分为多个仓库，可采用以下组合：

```text
snapmatch-web        # 官方网站
snapmatch-admin      # 管理后台
```

或：

```text
snapmatch-dashboard
snapmatch-console
snapmatch-backoffice
```

---

## 六、中文拼音命名方案（不推荐但可选）

若非常希望保留中文品牌原名，可使用拼音形式：

```text
yipaijihe-platform
yipaijihe-web
yipai-studio
```

⚠️ 注意：

- 在 GitHub / 招聘 / 技术传播场景中，英文命名体验明显更好
- 拼音不利于国际化与品牌识别

---

## 七、最终推荐（直接采用）

如果希望**不再纠结，快速定稿**，推荐：

```text
snapmatch-platform
```

理由：

- 品牌与技术边界清晰
- 可承载官网 + 后台 + 后续服务
- 长期可维护、可扩展
